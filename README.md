# - Реляционная база данных онлайн-школы «СмартВек»

Цель проекта — консолидация маркетинговых метрик верхнего уровня воронки, транзакционных данных учебного цикла и продуктовых логов вовлеченности студентов для построения системы сквозной аналитики и юнит-экономики.
Проект решает ключевую бизнес-задачу: устранение разрозненности данных между рекламными кабинетами, CRM-системой и внутренней LMS-платформой обучения. 
База данных спроектирована в SQLite DBeever и состоит из 5 консистентных таблиц, связанных между собой связями типа one-to-many (один ко многим):

1. Справочник пользователей (users) — 300 записей.
Содержит демографические и маркерные атрибуты клиентов на момент регистрации.
user_id (INTEGER, Primary Key) — уникальный идентификатор студента.
Дата регистрации (DATE) — временная отметка для формирования когорт.
Город (VARCHAR) / Возраст (INTEGER) — социально-демографические маркеры.
Email / Телефон (VARCHAR) — уникальные контакты для интеграции с CRM.chanel_id (INTEGER, Foreign Key) — идентификатор статического маркетингового канала привлечения.
  
2. Справочник номенклатуры курсов (courses) — 30 записей.
Продуктовая матрица школы, разделенная по ИТ-направлениям.
course_id (INTEGER, Primary Key) — уникальный код курса (смысловое кодирование: 100 — IT, 200 — Маркетинг, 300 — Дизайн).
Название курса (VARCHAR) — коммерческое наименование продукта.Категория (VARCHAR) — укрупненное продуктовое направление.
Цена (INTEGER) — базовая стоимость обучения для клиента.
Себестоимость (INTEGER) — внутренние издержки на производство и продажу курса.
Налог (FLOAT) — налоговая ставка (зафиксирована на уровне 0.06 для УСН «Доходы»).
  
3. Таблица фактов продаж (orders) — 600 записей.
Центральная транзакционная таблица, фиксирующая e-commerce цикл.
order_id (INTEGER, Primary Key) — уникальный номер транзакции.
user_id (INTEGER, Foreign Key) — связь со справочником студентов.
course_id (INTEGER, Foreign Key) — связь со справочником курсов.
Дата заказа (DATE) — дата совершения операции.
Статус (VARCHAR) — стадийность транзакции (Оплачен, Отменен, Возврат).
Промокод (VARCHAR) / Скидка (FLOAT) — примененные маркетинговые инструменты.
Финальная цена (INTEGER) — вычисляемая фактическая стоимость продажи с учетом дисконтирования.

4. Таблица маркетинговых расходов (marketing_costs).
Временной лог затрат на привлечение трафика в разрезе каналов.
Дата (DATE) — день фиксации расходов.
chanel_id (INTEGER, Foreign Key) — связь со справочником каналов.
Расходы (INTEGER) — абсолютные затраты на рекламную кампанию (Ad Spend).
Показы (INTEGER) / Клики (INTEGER) — метрики эффективности рекламных креативов.
  
5. Таблица продуктовой активности (activity).
Лог поведенческих метрик студентов внутри личного кабинета LMS-платформы.
user_id (INTEGER, Foreign Key) — идентификатор студента.
Дата посещения (DATE) — дата сессии пользователя.
Завершенные уроки (INTEGER) — кумулятивный прогресс студента по курсу.
Баллы (INTEGER) — показатель успеваемости по результатам тестирования.

Структура разработанной базы данных позволяет решать следующие аналитические задачи с помощью SQL-запросов и BI-инструментов.
Инструменты проектирования: DBeever, Miro, Яндекс Таблицы. 
Анализ и прототипирование: Excel, AW BI. 
Продуктовое прототипирование: Visily AI.
